SSH采用非对称加密，没有CA，由用户自己评估风险。 ## 密码登录 步骤： 1. client请求登录server。 1. server把自己的公钥发给client，client端第一次会显示此公钥的128位指纹以供用户评估核对，若同意则保存在`~/.ssh/known_hosts`中。 1. client使用这个公钥加密登录密码发回给server。 1. server用自己的私钥解密并验证登录密码，若正确则同意client登录。 ## 公钥登录（免密码登录） 步骤： 1. client请求登录server。 1. server发回一段随机字符串给client。 1. client用自己的私钥加密此字符串后发给server。 1. server用client事先保存在server上的公钥进行解密验证，若正确则同意登录。 client必须生成自己的公钥私钥对，并把公钥保存到server上。 生成：`ssh-keygen -t ed25519 -C "you@example.com"`并一路回车，生成的公钥`~/.ssh/id_rsa.pub`，私钥`~/.ssh/id_rsa`。 保存公钥到server：`ssh-copy-id user@server`，公钥会被保存到server的`~/.ssh/authorized_keys`中。等价命令`cat ~/.ssh/id_rsa.pub | ssh user@server 'cat >> ~/.ssh/authorized_keys'`。 ### GitHub 如果用[ssh登录GitHub](https://docs.github.com/en/authentication/connecting-to-github-with-ssh)等网站，则需要将公钥复制到保存网站上。 验证：`ssh -T git@github.com` 如果22端口被限制访问，可以使用HTTPS的443端口。 验证：`ssh -T -p 443 git@ssh.github.com` 要对所有相关连接启用，在用户配置里加入： ``` Host github.com Hostname ssh.github.com Port 443 ``` ### 验证代理`ssh-agent` 默认私钥`~/.ssh/id_rsa`在创建时会被自动加入验证代理或执行`ssh-add`。 有时需要通过跳转服务器无密码登录另一个服务器时，需要在本机上加入跳转服务器上的私钥`ssh-add <(ssh jump-server 'cat .ssh/id_rsa')`。 `ssh-add -l` 查看生效的公钥指纹 `ssh-add -L` 查看生效的公钥内容 `ssh-keygen -lf ~/.ssh/id_rsa.pub` 显示公钥的长度，指纹及注释 ## SSH Tunnel ### 本地端口转发 `local$ ssh -L 8080:target:80 middle` (-L [本地地址:]本地端口:目标主机:目标主机端口) 在本机上监听8080端口，并通过middle转发到target的80端口上。 e.g.: 本机上通过curl访问 `curl http://localhost:8080/index.html` `local$ ssh -L 8080:localhost:80 middle` 当target是localhost，即middle就是目标主机，将local的8080端口映射到middle的80端口。 ### 远程端口转发 `middle$ ssh -R 8080:target:80 local` (-R [远程地址:]远程主机端口:目标主机:目标主机端口) 在中间主机上执行，通过local（对于middle是远程主机）监听8080端口，并通过自己转发到target的80端口上。 要求local也运行有sshd。 `middle$ ssh -R 8080:localhost:80 local` 当target是localhost，即middle就是目标主机，将local的8080端口映射到middle的80端口。 ### 动态转发(SOCKS代理) `ssh -D 8080 server` 目标地址:目标端口则是不确定，而由发起的请求决定。 在本地监听8080端口，把数据转发到server上，server充当一个SOCKS代理服务器负责和外部主机通信。 e.g.: `curl --socks5 127.0.0.1:8080 www.google.com:443` 即由 server 请求 `www.google.com:443` ### X转发 DISPLAY环境变量: `DISPLAY=host:NumA.NumB` host: Xserver的地址, 如果为空表示Xserver运行于本机（一般为`DISPLAY=:0.0`，Xclient用unix domain socket而不是TCP连接到Xserver），如果是localhost指本机使用TCP方式。 NumA: 当图形程序(Xclient)使用TCP方式连接时, NumA为连接的端口减去6000的值, 如果NumA为0, 则表示连接到6000端口; 使用unix domain socket方式连接时, NumA则表示连接的unix socket的路径, 如果为0, 则表示连接到`/tmp/.X11-unix/X0`。 NumB: 几乎总为0. X转发配置: 服务端 `/etc/ssh/sshd_config`: X11Forwarding yes 客户端 `~/.ssh/config`: ForwardX11 yes （或直接`ssh -X`） 当从客户端ssh到服务端后会自动在服务端设置DISPLAY环境变量`DISPLAY=localhost:NumA.0`，服务端sshd会运行一个Xserver的模拟器并监听本地的NumA+6000端口，并将接收到的X命令通过ssh连接转到客户端，由客户端再作为Xclient发送显示在本地Xserver上（所以SSH客户端不需要运行`xhost +`）。 `xhost`: 控制Xserver访问权限。 为了安全，Xserver默认情况下不允许别的user(比如su切换时)或者别的host的图形程序显示在当前屏幕上，`xhost +[user/host]`命令将允许显示。 ### 常用参数 `-N` 只连接，不打开shell。 `-T` 不为这个连接分配TTY。 `-f` 连接成功后，转入后台运行。 `-NTf` 这个三个参数常一起使用，表示这个SSH连接只用来传数据，不需执行操作。 ## 远程操作 `tar czv dir | ssh server 'tar xz'` 复制dir目录到server `ssh server 'tar cz dir' | tar xzv` 复制server dir目录到本地 `ssh server 'ps auxf'` 显示服务器进程信息 ## 配置文件 server`sshd`配置文件：`/etc/ssh/sshd_config` （安装：`apt-get install openssh-server`）注意会被 `Include` 的配置覆盖 client用户配置文件：`~/.ssh/config`，全局：`/etc/ssh/ssh_config` ``` # 多条连接共享 ControlMaster auto ControlPath /tmp/ssh_mux_%h_%p_%r # 长连接, useful for scp ControlPersist 4h # 连接中转认证 ForwardAgent yes Host * # 保持连接，可能还需修改 server timeout: `/etc/profile.d/tmout.sh` ServerAliveInterval 240 # 减少延迟 GSSAPIAuthentication no GSSAPIDelegateCredentials yes # 加速连接 Ciphers arcfour Host myhost User kelvin HostName 192.168.2.100 Port 8888 ForwardX11 yes # 服务器间跳转 ProxyCommand ssh -q -W %h:%p 192.168.1.1 #ProxyCommand ssh 192.168.1.1 nc %h:%p #ProxyCommand ssh kelvin@192.168.1.1 "/usr/bin/bash -c 'exec 3<>/dev/tcp/%h/%p; cat <&3 & cat >&3;kill $!'" Host github.com HostName github.com User git # sock5 proxy ProxyCommand nc -v -x 127.0.0.1:1080 %h %p # http proxy ProxyCommand socat - PROXY:127.0.0.1:%h:%p,proxyport=3128 # http proxy ProxyCommand corkscrew 127.0.0.1 3128 %h %p # 通配 Host www* mail intranet HostName %h.example.com # 端口转发 Host api LocalForward 8080 localhost:80 ``` [更多选项](http://man7.org/linux/man-pages/man5/ssh_config.5.html) 跳转还可以直接：（jumper上设置启用 AllowTcpForwarding & AllowAgentForwarding） `ssh -J jumper target` ## Trouble Shooting 加上参数`ssh -vvv user@server`可以输出更多debug信息。 权限错误`WARNING: UNPROTECTED XXX FILE!`，需设置正确的文件权限： `~/.ssh/id_rsa` 600 `~/.ssh/id_rsa.pub` 600 `~/.ssh/known_hosts` 644 `~/.ssh` 755 检查log: /var/log/auth.log ## sshfs 把远程主机的文件系统通过ssh映射到本地。 挂载：`sshfs user@host:path/to/dir/ /local/mount/point` 取消：`fusermount -u /local/mount/point` [开机自动挂载](http://www.debuntu.org/mounting-a-fuse-filesystem-from-etcfstab/)：在`/etc/fstab`中加入`sshfs#user@host:path/to/dir/ /local/mount/point fuse defaults,auto,allow_other 0 0`，需要公钥登录。 另可用scp直接访问远程文件：`vim scp://user@host/path/to/file` ## [sshuttle](https://github.com/sshuttle/sshuttle) ## Reference [SSH, The Secure Shell: The Definitive Guide](http://docstore.mik.ua/orelly/networking_2ndEd/ssh/index.htm) [实战SSH端口转发](http://www.ibm.com/developerworks/cn/linux/l-cn-sshforward/)
